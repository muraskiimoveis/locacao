* HTML2PDF v3.10a *

   (25/08/2008)

ATTENTION :
----------
 - les unités ont été revues et corrigés depuis la version 2.70, vous devrez peut-etre corriger vos HTML.
   Unités mm, in, pt, px réellement reproduites pour toutes les propriétés (dont font-size) tel que 25.4mm = 1in, 72pt = 1in, 96px = 1in
   
 - les propriétés de la balise PAGE ont été modifiées depuis la version 3.04 afin de rajouter plus d'options, regardez l'exemple 7
 
 - toutes les balises ne sont pas encore codées (comme par exemple UL, LI, OL, DIV, P....)
 
Utilisation :
------------
 - regardez les exemples fournis pour voir le fonctionnement.
 
 - il est TRES IMPORTANT de fournir du HTML 4.01 valide au convertisseur, mais seulement ce que contient le <body>
       
 - pour les borders : il est conseillé qu'ils soient au format "solid 1mm #000000"

 - pour les paddings : ils ne sont applicables qu'aux balises table, th, td

 - la balise <page></page> a été rajoutée. CF Exemple 7.
   Elle permet de définir l'orientation, les marges top et bottom, l'image de fond d'une page, sa taille et position, le footer.
   Il est egalement possible de garder les header et footer des pages précédentes, grace à l'attribue pageset="old", CF Exemple 3
 
 - les balises <page_header></page_header> et <page_footer></page_footer> ont été rajoutées, CF Exemple 3
  
 - la balise <nobreak></nobreak> a été rajoutée afin de forcer l'affichage d'une partie sur une même page.
   Si cette partie ne rentre pas dans le reste de la page, un saut de page est effectué avant.
 
 - la balise <barcode></barcode> a été rajoutée afin de pouvoir inserer des barcodes dans les pdfs, CF Exemples 0 et 9
   Les types de codebar possible sont : EAN13, UPC_A, CODE39
 
 - la liste des balises HTML reconnues se trouve dans le fichier "_balises_html.xls"

 - la possibilité de protégé vos PDF a été rajouté, CF Exemple 7. Elle utilise le script fpdf_protection de Klemen Vodopivec.
